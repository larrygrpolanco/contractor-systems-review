## The Payment Request Process (What This Folder Reveals)

To collect payment on a City of Tampa housing rehab contract, Lionize Construction must assemble and submit:

1. **Invoice** — itemized, addressed to City of Tampa Development & Economic Opportunity
2. **Contractor Payment Request Form** — signed by contractor AND homeowner
3. **Cost Breakdown** — itemized expenses with vendor names, dates, check numbers
4. **DMI-30 Form** — subcontractor payment report (see Operating Procedure Manual folder)
5. **Final Payment Affidavit & Release** — notarized, certifying all subs are paid
6. **Consolidated Lien Release** — signed by every subcontractor
7. **Warranty Acknowledgement** — specifying warranty periods for each trade

The `To Do to submmit payment Request to Tampa.docx` checklist is the internal procedure guide for assembling this package.

## Pain Points Observed

- The entire payment package for one job spans 12+ separate PDF files with inconsistent naming — easy to miss a document or submit an outdated version
- Multiple duplicates of the same form exist (two affidavit files, two lien release files) — unclear which is the current/correct version
- The "To Do" checklist is a Word doc that has to be manually consulted — no automated reminders that steps are complete or outstanding
- Collecting lien releases from all subs requires coordinating 5+ separate companies — a bottleneck with no tracking mechanism
- Warranty dates are tracked only in this one PDF — no calendar reminders when warranties expire
- The `To Do` filename has a typo ("submmit") — suggests files are named quickly under pressure, making future searching harder

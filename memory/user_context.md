The user is doing systems consulting / workflow improvement work for a small contracting business (Lionize Construction / Luz B. Polanco). They want to understand existing documents and processes before recommending or building solutions. They mentioned interest in Claude Code skills as part of the solution.

The work is practical and operational — not software engineering in the traditional sense, but process documentation, workflow design, and potentially light automation (form pre-filling, checklists, reminders). Keep recommendations grounded and achievable for a non-technical one-person business.

So adoption risk lands on the user, not Luz — this is why Notion is acceptable despite Luz being non-technical.

**Chosen direction:** Notion as the tracker/dashboard (user-maintained) + Dropbox as the file store (Luz already keeps everything there — organized-ish but not great) + Claude skills as a "cowork" layer that walks Luz through each process from her uploaded files. User is still designing the skill-based system where Luz uploads project files and a Claude skill guides her through. CompanyCam was considered but deferred — it solves photo documentation, not the paperwork/compliance/payment bottleneck that is the real pain.
